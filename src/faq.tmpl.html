A simple FAQ for all of your needs!

What is this website?

The Windmill is an interactive alternative to pen, graph paper, and a camera for creating and sharing puzzles using the puzzle language of the 2011 2016 video game The Witness. Anyone can play and build puzzles and anyone can share puzzles with the world. I am not associated with Thekla Inc and this website uses no assets or code from the game.

Check out /r/thewindmill on Reddit to find changelogs and various useful links.

This app was originally called The Witness Puzzle Simulator and it was just a static website. That website now redirects here.

Is this website a video game?

It is a tool for The Witness community. If you want to learn the puzzle mechanics, I'd strongly encourage you to play The Witness. The way they interact with the game world is in service of ideas which are really awe-inspiring.

How does this website work?

It is an AngularJS app with persistent server state. When building puzzles, all puzzle state can be encoded in the URL. This means that you can keep a tab open and checkpoint offline.

Currently, anyone with a Google account login can publish puzzles. I am excited to see what you come up with!

What are known issues?

As far as missing/broken puzzle mechanics: first, yellow polyominos and blue polyominos cancel out based on count, not on whether they fit over each other. Second, blue/yellow configurations that require the yellow shapes to lie outside of the grid sometimes validate and sometimes don't. Third, Ys and stars/suns interact in ways incompatible with some game puzzles. Fourth, polyomino search is done differently when there is a Y, but cannot yet handle multiple Ys to cancel out multiple polyominos. This is roughly in order of heavy to light work required to fix.

Outside of the puzzle logic I can guarantee there's more. See next section for how to report them!

Are you going to add any more features?

Yes! The best way to suggest features, or report bugs, or write code to implement features or fix bugs, is to use GitHub issue tracking, which requires a GitHub account.

For something more lightweight, send suggestions using this form or directly to <windmill> at <this website without the subdomain>.

Some feature requests which have been requested and will be implemented include: collections of puzzles to play in a row, better play (arrow keys, click to draw on mobile, adjust better to screen size), better editing (keyboard shortcuts, drag to add multiple of the same thing, easier grid size changes), generation of close-to-uniquely solvable puzzles.
